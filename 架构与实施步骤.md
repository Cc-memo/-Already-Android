# 酒店房型爬取 - 架构与实施步骤

本文档描述「云端任务 + 本地调度 + 手机执行」的部署架构，以及按此架构实施的步骤。换到其他对话或仓库时，可说：**请按本项目的 `架构与实施步骤.md` 来做。**

---

## 一、架构图

![架构图：云端 → 本地控制服务 → 多执行端](assets/architecture_cloud_local_full.png)

- **云端**：任务库、数据库（指令+结果）、API 服务；负责下发任务、接收上报结果。
- **本地控制服务**：跑在本地 PC/服务器；从云端领任务、下发给执行端、多设备/多账号调度、结果汇总上报。
- **执行端（多端）**：多台手机/不同账户；执行端① 携程 1/2/3.py、执行端② 美团 Meituan、执行端③ 飞猪等。

**为何需要本地控制服务**：云服务器无法直接连接本地手机（无公网 IP / USB 连在电脑上），因此必须由「能同时连云端和手机」的本地电脑做中转：主动向云端拉任务 → 在本地派给手机执行 → 再把结果推回云端。

## 二、三端与脚本对应关系

| 平台 | 执行端脚本/目录 | 说明 |
|------|------------------|------|
| 携程 | `3.py`（全流程）、`1.py`（解析）、`2.py`（Inspector） | 房型在 WebView，用 1.py 解析 UI dump |
| 美团 | `Meituan/meituan_extract.py`、`Meituan/meituan_navigate.py` | 可 `--device` 多屏抓取，输出 Meituan/1.json |
| 飞猪 | `Feizhu/feizhu_api_to_json.py`（抓包接口）、或不用抓包方案 | 见 Feizhu/抓包接口方案.md |

## 三、实施步骤（建议顺序）

### 第一步：云端准备（任务 + 结果存储）

1. **Clone hotel-crawler-backup**
   - 在**要跑 API 和 MySQL 的那台机器**上执行：`git clone https://github.com/Cc-memo/hotel-crawler-backup.git`
   - **有云服务器** → 在云上 clone，在云上建库、跑 API。
   - **没有云 / 先本地联调** → 在本地电脑 clone（可与本仓库同级目录），本机起 MySQL + API。
2. **部署 MySQL**
   - 按 `hotel-crawler-backup/Rpa/database/` 下说明（如 `setup_mysql.py`、`数据库表结构说明.md`）建库建表。
3. **任务表**
   - 至少包含：任务 ID、平台（携程/美团/飞猪）、酒店标识、入离店日期、状态（待执行/执行中/已完成/失败），可选 `device_id`、`account` 以支持多机多账号。
4. **API**
   - **领任务**：GET，返回待执行任务并标记为执行中（可按 platform、device_id、account 筛选）。
   - **上报结果**：POST，接收 task_id、结果 JSON（如 1.json 内容），写结果表并标记任务完成。

### 第二步：本地调度（在连手机的电脑上）

1. **新建「本地调度」脚本/服务**（建议 Python）
   - 循环（或定时）调云端「领任务」→ 有任务则按 `platform` 调对应脚本 → 执行完后调「上报结果」。
2. **按平台调用**
   - 携程：用任务参数调 `3.py`，传 `device_id`（ADB 序列号）若支持。
   - 美团：调 `Meituan/meituan_extract.py`（如 `--device`）或 navigate。
   - 飞猪：按 Feizhu 抓包/接口方案调 `Feizhu/feizhu_api_to_json.py` 或完整流程。
3. **结果回传**
   - 读取本地产出的 `1.json`（或 Meituan/1.json、Feizhu/1.json），作为 body 调「上报结果」API。

### 第三步：单机单手机跑通

- 云端录入 1～2 条测试任务 → 本地只连一台手机跑本地调度 → 确认任务完成、结果在云端正确。

### 第四步：扩展（多机 / 多账号）

- 领任务时带 `device_id`，本地按设备分配；任务或配置带 `account`，支持多账号；加超时、失败重试、任务回滚为待执行。

## 四、Clone 与仓库位置小结

| 场景 | Clone 位置 | 说明 |
|------|------------|------|
| 有云服务器 | **云服务器** | 在云上 clone hotel-crawler-backup，云上建库、跑 API；本地只跑「本地调度」+ 本仓库（room_rules_isolated）。 |
| 无云 / 先本地试 | **本地电脑** | 本机 clone hotel-crawler-backup，本机 MySQL + API；与本仓库同机，方便联调。 |
