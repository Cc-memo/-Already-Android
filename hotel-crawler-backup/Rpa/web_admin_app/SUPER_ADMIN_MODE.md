# 超级管理员模式文档

## 概述

系统采用**超级管理员模式**进行权限管理，实现了更细粒度的权限控制，确保系统安全性和管理灵活性。

## 角色体系

系统包含四种角色，按权限从低到高排列：

| 角色 | 角色值 | 权限说明 |
|------|--------|---------|
| 普通用户 | `user` | 只能查看和操作自己的数据 |
| 操作员 | `operator` | 可以创建和管理爬取任务 |
| 管理员 | `admin` | 可以管理普通用户和操作员 |
| 超级管理员 | `super_admin` | 拥有所有权限，可以管理所有用户（包括其他管理员） |

## 核心规则

### 1. 管理员管理限制

- **普通管理员**只能管理普通用户和操作员
- **普通管理员**不能创建、编辑、删除或禁用其他管理员
- **超级管理员**可以管理所有用户，包括其他管理员和超级管理员

### 2. 自我保护规则

所有用户（包括超级管理员）都不能：
- 删除自己的账户
- 禁用自己的账户
- 降级自己的角色（超级管理员可以，但不推荐）

### 3. 最后一个管理员保护

系统会保护最后一个管理员（包括超级管理员）：
- 不能删除最后一个管理员
- 不能禁用最后一个管理员
- 不能将最后一个管理员降级

## 使用场景

### 场景1：创建超级管理员

**方式1：通过数据库直接创建**

```sql
-- 查看现有用户
SELECT id, username, role FROM users;

-- 将某个用户设置为超级管理员
UPDATE users SET role = 'super_admin' WHERE username = 'admin';
```

**方式2：通过超级管理员账户创建**

1. 使用超级管理员账户登录
2. 进入用户管理页面
3. 点击"创建用户"
4. 在角色下拉框中选择"超级管理员"

### 场景2：普通管理员管理用户

普通管理员登录后：
- ✅ 可以查看所有用户列表（包括管理员，但无法操作）
- ✅ 可以创建、编辑、删除普通用户和操作员
- ❌ 不能创建、编辑、删除管理员
- ❌ 看到管理员用户时，操作按钮显示"无权限"

### 场景3：超级管理员管理所有用户

超级管理员登录后：
- ✅ 可以查看所有用户列表
- ✅ 可以创建、编辑、删除所有角色用户
- ✅ 可以管理其他管理员和超级管理员
- ✅ 角色下拉框显示所有选项（包括超级管理员）

## API 接口说明

### 权限检查函数

```python
def _can_manage_user(operator: Dict, target_user: Dict) -> tuple[bool, str]:
    """
    检查操作者是否可以管理目标用户
    返回: (是否可以管理, 错误信息)
    """
```

**逻辑：**
- 超级管理员可以管理所有用户
- 普通管理员只能管理普通用户和操作员
- 其他角色不能管理用户

### 用户管理 API

所有用户管理 API 都需要管理员或超级管理员权限：

- `GET /api/auth/users` - 获取用户列表
- `GET /api/auth/users/<id>` - 获取用户详情
- `POST /api/auth/users` - 创建用户（权限检查：普通管理员不能创建管理员）
- `PUT /api/auth/users/<id>` - 更新用户（权限检查：普通管理员不能编辑管理员）
- `DELETE /api/auth/users/<id>` - 删除用户（权限检查：普通管理员不能删除管理员）
- `POST /api/auth/users/<id>/toggle-status` - 切换状态（权限检查：普通管理员不能禁用管理员）
- `POST /api/auth/users/<id>/reset-password` - 重置密码

## 前端实现

### 角色显示

- **超级管理员**：紫色背景，白色文字
- **管理员**：蓝色背景，蓝色文字
- **操作员**：橙色背景，橙色文字
- **普通用户**：灰色背景，灰色文字

### 操作按钮显示逻辑

1. **当前用户**：只能编辑和重置密码
2. **管理员用户（普通管理员视角）**：显示"无权限"，按钮禁用
3. **管理员用户（超级管理员视角）**：可以正常操作
4. **普通用户/操作员**：可以编辑、切换状态、删除

### 创建用户表单

- **超级管理员**：角色下拉框显示所有选项（包括超级管理员）
- **普通管理员**：角色下拉框只显示"普通用户"和"操作员"，并显示提示文字

### 编辑用户表单

- **超级管理员**：可以编辑所有用户的角色
- **普通管理员**：编辑管理员时，角色字段禁用，显示提示文字

## 数据库迁移

### 将现有管理员升级为超级管理员

```sql
-- 方式1：将所有管理员升级为超级管理员
UPDATE users SET role = 'super_admin' WHERE role = 'admin';

-- 方式2：只升级特定用户
UPDATE users SET role = 'super_admin' WHERE username = 'admin';
```

### 创建新的超级管理员

```sql
-- 假设已有用户表，直接更新角色
UPDATE users SET role = 'super_admin' WHERE username = 'your_username';
```

## 安全建议

1. **最少权限原则**：只将必要的用户设置为超级管理员
2. **定期审计**：定期检查超级管理员列表，确保只有授权人员
3. **密码策略**：超级管理员账户应使用强密码
4. **操作日志**：建议记录所有用户管理操作（未来功能）

## 常见问题

### Q1: 如何创建第一个超级管理员？

A: 如果系统中还没有超级管理员，可以通过数据库直接设置：

```sql
UPDATE users SET role = 'super_admin' WHERE username = 'admin' LIMIT 1;
```

### Q2: 普通管理员能看到其他管理员吗？

A: 可以。普通管理员可以看到所有用户列表，包括其他管理员，但无法对其进行操作（编辑、删除、禁用等）。

### Q3: 超级管理员可以删除自己吗？

A: 不可以。系统保护所有用户不能删除自己的账户，包括超级管理员。

### Q4: 如果所有管理员都被删除了怎么办？

A: 系统会保护最后一个管理员，不允许删除。如果确实需要重置，可以通过数据库直接操作。

### Q5: 如何区分超级管理员和普通管理员？

A: 
- 在用户列表中，超级管理员显示为紫色标签
- 在创建用户时，只有超级管理员可以看到"超级管理员"选项
- 在编辑用户时，普通管理员编辑管理员用户时，角色字段会禁用
