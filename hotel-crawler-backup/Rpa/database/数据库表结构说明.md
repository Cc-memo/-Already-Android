## 1. 用户表 (users)

**用途**: 存储系统用户信息，用于身份认证和权限管理

**位置**: `Rpa/web_admin_app/auth.py`

### SQLite 结构

```sql
CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL UNIQUE,
    password_hash TEXT NOT NULL,
    role TEXT NOT NULL DEFAULT 'user',
    created_at TEXT NOT NULL,
    last_login_at TEXT
)
```

### MySQL 结构

```sql
CREATE TABLE IF NOT EXISTS users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(20) NOT NULL DEFAULT 'user',
    created_at VARCHAR(30) NOT NULL,
    last_login_at VARCHAR(30)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
```

### 字段说明

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|---------|------|
| id | INTEGER/INT | 是 | 主键，自增 |
| username | TEXT/VARCHAR(100) | 是 | 用户名，唯一 |
| password_hash | TEXT/VARCHAR(255) | 是 | 密码哈希值 |
| role | TEXT/VARCHAR(20) | 是 | 用户角色，默认 'user' |
| created_at | TEXT/VARCHAR(30) | 是 | 创建时间 |
| last_login_at | TEXT/VARCHAR(30) | 否 | 最后登录时间 |

### 索引

- `username`: UNIQUE 索引

## 2. 爬取任务表 (crawl_tasks)

**用途**: 存储RPA爬取任务的信息，包括任务状态、进度、结果等

**位置**: `Rpa/web_admin_app/crawl_tasks.py`

```sql
CREATE TABLE IF NOT EXISTS crawl_tasks (
    task_id TEXT PRIMARY KEY,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    started_at TEXT,
    finished_at TEXT,
    status TEXT NOT NULL,
    hotel_name TEXT NOT NULL,
    location TEXT,
    check_in TEXT,
    check_out TEXT,
    platforms_json TEXT NOT NULL,
    progress INTEGER NOT NULL DEFAULT 0,
    current_platform TEXT,
    error TEXT,
    results_json TEXT,
    user_id INTEGER
)
```

### MySQL 结构

```sql
CREATE TABLE IF NOT EXISTS crawl_tasks (
    task_id VARCHAR(50) PRIMARY KEY,
    created_at VARCHAR(30) NOT NULL,
    updated_at VARCHAR(30) NOT NULL,
    started_at VARCHAR(30),

    status VARCHAR(20) NOT NULL,

    check_in VARCHAR(20),

    current_platform VARCHAR(50),
```

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|---------|------|
| task_id | TEXT/VARCHAR(50) | 是 | 主键，任务ID（UUID） |
| created_at | TEXT/VARCHAR(30) | 是 | 创建时间 |
| updated_at | TEXT/VARCHAR(30) | 是 | 更新时间 |

| status | TEXT/VARCHAR(20) | 是 | 任务状态：queued/running/completed/failed |

| location | TEXT/VARCHAR(255) | 否 | 位置/地址 |
| check_in | TEXT/VARCHAR(20) | 否 | 入住日期 |
| check_out | TEXT/VARCHAR(20) | 否 | 离店日期 |
| platforms_json | TEXT | 是 | 平台列表（JSON格式） |
| progress | INTEGER/INT | 是 | 进度（0-100），默认0 |

| error | TEXT | 否 | 错误信息 |

## 4. 搜索记录表 (search_records)

**用途**: 存储RPA搜索记录，记录每次搜索的基本信息

**位置**: `Rpa/database/db_utils.py`

### SQLite 结构

```sql
CREATE TABLE IF NOT EXISTS search_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    search_time TEXT NOT NULL,
    platform TEXT NOT NULL,
    address TEXT,
    hotel_keyword TEXT,
    hotel_name TEXT,
    room_count INTEGER DEFAULT 0,
    user_id INTEGER,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
)
```

### MySQL 结构

```sql
CREATE TABLE IF NOT EXISTS search_records (
    id INT AUTO_INCREMENT PRIMARY KEY,
    search_time VARCHAR(50) NOT NULL,
    platform VARCHAR(20) NOT NULL,
    address VARCHAR(255),
    hotel_keyword VARCHAR(255),
    hotel_name VARCHAR(255),
    room_count INT DEFAULT 0,
    user_id INT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_search_time (search_time),
    INDEX idx_platform (platform),
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
```

### 字段说明

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|---------|------|
| id | INTEGER/INT | 是 | 主键，自增 |
| search_time | TEXT/VARCHAR(50) | 是 | 搜索时间 |
| platform | TEXT/VARCHAR(20) | 是 | 平台名称（meituan/ctrip/fliggy/gaode） |
| address | TEXT/VARCHAR(255) | 否 | 搜索地址 |
| hotel_keyword | TEXT/VARCHAR(255) | 否 | 酒店关键词 |
| hotel_name | TEXT/VARCHAR(255) | 否 | 酒店名称 |
| room_count | INTEGER/INT | 否 | 房型数量，默认0 |
| user_id | INTEGER/INT | 否 | 用户ID（账户隔离） |
| created_at | TEXT/TIMESTAMP | 否 | 创建时间 |

### 索引

- `idx_search_time`: 搜索时间索引
- `idx_platform`: 平台索引
- `idx_user_id`: 用户ID索引（MySQL）

## 5. 房型数据表 (room_data)

**用途**: 存储搜索记录对应的房型详细信息，与 search_records 表关联

**位置**: `Rpa/database/db_utils.py`

### SQLite 结构

```sql
CREATE TABLE IF NOT EXISTS room_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    search_id INTEGER NOT NULL,
    room_name TEXT NOT NULL,
    price TEXT,
    remaining_rooms TEXT,
    remarks TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (search_id) REFERENCES search_records(id) ON DELETE CASCADE
)
```

### MySQL 结构

```sql
CREATE TABLE IF NOT EXISTS room_data (
    id INT AUTO_INCREMENT PRIMARY KEY,
    search_id INT NOT NULL,
    room_name VARCHAR(255) NOT NULL,
    price VARCHAR(50),
    remaining_rooms VARCHAR(50),
    remarks TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (search_id) REFERENCES search_records(id) ON DELETE CASCADE,
    INDEX idx_search_id (search_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
```

### 字段说明

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|---------|------|
| id | INTEGER/INT | 是 | 主键，自增 |
| search_id | INTEGER/INT | 是 | 外键，关联 search_records.id |
| room_name | TEXT/VARCHAR(255) | 是 | 房型名称 |
| price | TEXT/VARCHAR(50) | 否 | 价格 |
| remaining_rooms | TEXT/VARCHAR(50) | 否 | 剩余房间数 |
| remarks | TEXT | 否 | 备注信息 |
| created_at | TEXT/TIMESTAMP | 否 | 创建时间 |

### 索引

- `idx_search_id`: 搜索记录ID索引（MySQL）
- `search_id`: 外键约束

### 关联关系

- `room_data.search_id` → `search_records.id` (ON DELETE CASCADE)

## 6. 酒店数据表 (hotel_data) ⭐ 新建

**创建时间**: 2024-12-19

```sql
    hotel_url TEXT,
    star_level TEXT,
    rating_score REAL,
    review_count INTEGER,
    min_price REAL,
    booking_dynamic TEXT,
    address TEXT,
    region TEXT,
    opening_date TEXT,
    room_types TEXT,
    phone TEXT,
    email TEXT,
    website TEXT,
    crawl_time TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at TEXT DEFAULT CURRENT_TIMESTAMP
)
```

### MySQL 结构

```sql
CREATE TABLE IF NOT EXISTS hotel_data (
    id INT AUTO_INCREMENT PRIMARY KEY,
    hotel_name VARCHAR(200) NOT NULL,
    platform VARCHAR(50) NOT NULL,
    hotel_id VARCHAR(100),
    hotel_url VARCHAR(500),
    star_level VARCHAR(50),
    rating_score FLOAT,
    review_count INT,
    min_price FLOAT,
    booking_dynamic VARCHAR(200),
    address VARCHAR(500),
    region VARCHAR(100),
    opening_date VARCHAR(100),
    room_types JSON,
    phone VARCHAR(50),
    email VARCHAR(100),
    website VARCHAR(500),
    crawl_time DATETIME,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_hotel_platform (hotel_name, platform),
    INDEX idx_hotel_name (hotel_name),
    INDEX idx_platform (platform),
    INDEX idx_region (region),
    INDEX idx_crawl_time (crawl_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
```

### 字段说明

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|---------|------|
| id | INTEGER/INT | 是 | 主键，自增 |
| hotel_name | TEXT/VARCHAR(200) | 是 | 酒店名称 |
| platform | TEXT/VARCHAR(50) | 是 | 平台名称（美团/携程/飞猪/高德） |
| hotel_id | TEXT/VARCHAR(100) | 否 | 平台酒店ID |
| hotel_url | TEXT/VARCHAR(500) | 否 | 酒店URL |
| star_level | TEXT/VARCHAR(50) | 否 | 星级（如：五星、四星） |
| rating_score | REAL/FLOAT | 否 | 点评分数（0-5分） |
| review_count | INTEGER/INT | 否 | 点评条数 |
| min_price | REAL/FLOAT | 否 | 最低价格（起价） |
| booking_dynamic | TEXT/VARCHAR(200) | 否 | 预订动态（如：5分钟前有人预订） |
| address | TEXT/VARCHAR(500) | 否 | 酒店地址 |
| region | TEXT/VARCHAR(100) | 否 | 区域（城市-区县） |
| opening_date | TEXT/VARCHAR(100) | 否 | 开业时间 |
| room_types | TEXT/JSON | 否 | 房型信息（JSON格式或TEXT） |
| phone | TEXT/VARCHAR(50) | 否 | 联系电话 |
| email | TEXT/VARCHAR(100) | 否 | 邮箱 |
| website | TEXT/VARCHAR(500) | 否 | 官网 |
| crawl_time | TEXT/DATETIME | 否 | 爬取时间 |
| created_at | TEXT/TIMESTAMP | 否 | 创建时间 |
| updated_at | TEXT/TIMESTAMP | 否 | 更新时间 |

### 索引

- `uk_hotel_platform`: 唯一索引（hotel_name, platform）- 防止同一酒店在同一平台重复
- `idx_hotel_name`: 酒店名称索引
- `idx_platform`: 平台索引
- `idx_region`: 区域索引
- `idx_crawl_time`: 爬取时间索引

### 唯一约束

- `(hotel_name, platform)`: 确保同一酒店在同一平台只有一条记录，更新时使用 ON DUPLICATE KEY UPDATE（MySQL）或 INSERT OR REPLACE（SQLite）

### 使用说明

1. **数据保存**: 使用 `save_hotel_data_to_database()` 函数保存数据
2. **数据查询**: 使用 `/api/hotels` 接口查询数据，支持多条件筛选和分页
3. **数据详情**: 使用 `/api/hotels/<id>` 接口获取单条酒店详情

### 相关函数

- `save_hotel_data_to_database()`: 保存酒店数据到数据库
- `query_hotel_data()`: 查询酒店数据（支持多条件筛选、分页、排序）

### 相关接口

- `GET /api/hotels`: 查询酒店数据列表
- `GET /api/hotels/<id>`: 获取酒店详情

## 数据库类型支持

系统支持两种数据库类型：

1. **SQLite**: 本地数据库，文件存储在 `Rpa/database/` 目录
2. **MySQL**: 云数据库，通过配置文件或环境变量配置

数据库类型通过 `db_config.json` 或环境变量 `DB_TYPE` 配置。

## 注意事项

1. **唯一约束**: `hotel_data` 表的 `(hotel_name, platform)` 组合必须唯一，插入重复数据时会自动更新
2. **外键约束**: `room_data` 表的外键设置为级联删除，删除 `search_records` 记录时会自动删除关联的 `room_data` 记录
3. **账户隔离**: `search_records` 和 `crawl_tasks` 表支持 `user_id` 字段，实现多用户数据隔离
4. **JSON字段**: MySQL 的 `hotel_data.room_types` 使用 JSON 类型，SQLite 使用 TEXT 类型存储 JSON 字符串
