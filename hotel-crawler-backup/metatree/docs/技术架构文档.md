# 酒店信息爬取系统 - 技术架构文档

## 一、架构概述

### 1.1 架构设计理念

本系统采用 **LangGraph** 框架构建爬取流程，将复杂的爬取任务分解为多个角色（Agent）协作完成。每个平台（美团、携程、飞猪、高德）拥有独立的流程图（Graph），通过状态机管理爬取流程，实现高度模块化、可扩展、易维护的爬取架构。

### 1.2 核心优势

- **模块化设计**：每个角色职责单一，易于测试和维护
- **流程可视化**：LangGraph 提供流程可视化，便于理解和调试
- **状态管理**：通过状态机管理爬取流程，支持暂停、恢复、重试
- **平台隔离**：每个平台独立流程，互不干扰
- **可扩展性**：新增平台只需定义新的流程图
- **容错性**：每个节点可独立处理异常，支持流程回退

### 2.2 状态定义

每个平台的爬取流程共享统一的状态结构：

```python
@dataclass
class CrawlState:
    """爬取状态"""
    # 任务信息
    task_id: str
    hotel_name: str
    platform: str
    
    # 流程状态
    current_node: str  # 当前节点
    visited_nodes: List[str]  # 已访问节点
    error_count: int  # 错误计数
    
    # 数据
    urls: Dict[str, str]  # 存储的URL（base_url, search_url, detail_url等）
    login_status: bool  # 登录状态
    search_results: List[Dict]  # 搜索结果
    hotel_data: List[Dict]  # 酒店数据
    validation_results: Dict[str, Any]  # 校验结果
    
    # 元数据
    start_time: datetime
    end_time: Optional[datetime]
    logs: List[str]  # 日志记录
```

### 2.3 角色定义

#### 2.3.1 网址获取角色 (URLFetcherAgent)

**职责**：
- 从配置文件获取平台基础URL
- 构建搜索URL和详情页URL
- 验证URL有效性

**输入**：平台名称、酒店名称
**输出**：URL字典（base_url, search_url, detail_url）

**节点名称**：`fetch_urls`

#### 2.3.2 登入角色 (LoginAgent)

**职责**：
- 检测是否需要登录
- 执行登录流程
- 验证登录状态
- 处理验证码（如需要）

**输入**：URL、用户名、密码、登录配置
**输出**：登录状态、Cookie/Session

**节点名称**：`login`

#### 2.3.3 定位角色 (LocatorAgent)

**职责**：
- 定位搜索输入框
- 定位搜索按钮
- 定位结果列表容器
- 定位酒店详情元素

**输入**：页面HTML、XPath配置
**输出**：元素定位结果、定位的XPath

**节点名称**：`locate_elements`

#### 2.3.4 酒店查找角色 (SearchAgent)

**职责**：
- 输入酒店名称
- 执行搜索操作
- 等待搜索结果加载
- 检测结果页面状态

**输入**：酒店名称、搜索配置
**输出**：搜索结果列表

**节点名称**：`search_hotel`

#### 2.3.5 内容校验角色 (ValidatorAgent)

**职责**：
- 校验数据完整性
- 校验数据格式
- 校验数据有效性
- 生成校验报告

**输入**：提取的数据
**输出**：校验结果、校验报告

**节点名称**：`validate_content`

#### 2.3.6 数据提取角色 (ExtractorAgent)

**职责**：
- 提取酒店列表数据
- 提取酒店详情数据
- 数据清洗和格式化
- 数据标准化

**输入**：页面HTML、提取配置
**输出**：结构化数据

**节点名称**：`extract_data`

#### 2.3.7 错误处理角色 (ErrorHandlerAgent)

**职责**：
- 捕获和处理异常
- 记录错误日志
- 决定重试或跳过
- 生成错误报告

**输入**：异常信息、当前状态
**输出**：处理决策（retry/skip/fail）

**节点名称**：`handle_error`

## 三、平台流程设计

### 3.1 美团平台流程

```python
# 流程节点顺序
1. fetch_urls (网址获取)
   ↓
2. login (登入) [条件：需要登录]
   ↓
3. locate_elements (定位元素)
   ↓
4. search_hotel (酒店查找)
   ↓
5. extract_data (数据提取)
   ↓
6. validate_content (内容校验)
   ↓
7. END (完成)
```

**条件边**：
- `fetch_urls` → `login`: 如果配置了用户名密码
- `login` → `locate_elements`: 登录成功或不需要登录
- `validate_content` → `extract_data`: 校验失败时回退
- `validate_content` → `END`: 校验成功

## 四、节点实现规范

### 4.1 节点函数签名

```python
def node_function(state: CrawlState) -> CrawlState:
    """
    节点函数标准签名
    
    Args:
        state: 当前爬取状态
        
    Returns:
        更新后的状态
    """
    # 1. 记录开始
    state.logs.append(f"[{state.current_node}] 开始执行")
    
    try:
        # 2. 执行节点逻辑
        result = execute_node_logic(state)
        
        # 3. 更新状态
        state.current_node = "next_node"
        state.visited_nodes.append(state.current_node)
        
        # 4. 记录成功
        state.logs.append(f"[{state.current_node}] 执行成功")
        
    except Exception as e:
        # 5. 错误处理
        state.error_count += 1
        state.logs.append(f"[{state.current_node}] 执行失败: {e}")
        # 路由到错误处理节点
        state.current_node = "handle_error"
    
    return state
```

### 4.2 条件边函数

```python
def should_login(state: CrawlState) -> str:
    """
    判断是否需要登录
    
    Returns:
        "login" - 需要登录
        "locate_elements" - 不需要登录
    """
    if state.urls.get("username") and state.urls.get("password"):
        return "login"
    return "locate_elements"
```

### 4.3 错误处理策略

```python
def error_handler(state: CrawlState) -> str:
    """
    错误处理决策
    
    Returns:
        "retry" - 重试当前节点
        "skip" - 跳过当前节点
        "fail" - 任务失败
    """
    if state.error_count < 3:
        return "retry"
    elif state.error_count < 5:
        return "skip"
    else:
        return "fail"
```
