# 酒店信息爬取及管理平台 - 完整需求文档

## 一、项目背景

随着旅游行业的蓬勃发展，用户对于酒店信息的需求日益多样化，需要便捷地获取不同平台（美团、携程、飞猪、高德）上酒店的全面信息，以便进行对比和选择。同时，企业也需要一个高效的管理平台对爬取到的酒店信息进行统一管理、分析和应用。为满足这一需求，特规划开发本酒店信息爬取及管理平台，实现对主流平台酒店信息的精准爬取与高效管理。

## 二、产品目标

1. 实现对美团、携程、飞猪、高德四大平台酒店信息的自动化爬取，涵盖搜索列表页和酒店详情页指定字段信息。
2. 搭建功能完善的管理平台，支持对爬取数据的存储、查询、展示、分析、导出等操作，为用户提供全面的酒店信息管理服务。
3. 保证爬取数据的准确性、及时性和完整性，确保管理平台运行稳定、操作便捷，满足用户对酒店信息管理的多样化需求。

## 三、产品范围

1. **信息爬取范围**：覆盖美团、携程、飞猪、高德四大主流酒店预订平台，针对用户输入的酒店名称，爬取该酒店在各平台的搜索列表页和详情页指定字段信息。
2. **管理平台功能范围**：包括数据爬取管理、数据存储与管理、数据查询与展示、数据分析、数据导出、用户管理、系统设置等模块。

## 四、用户角色分析

### 4.1 用户角色定义

#### 角色1：系统管理员（Admin）
- **职责**：系统整体管理、用户管理、系统配置
- **权限**：所有功能权限
- **主要功能**：
  - 用户管理（创建、编辑、禁用用户）
  - 角色权限配置
  - 系统设置（爬取参数、存储策略、日志管理）
  - 数据爬取管理
  - 数据查询与分析
  - 数据导出

#### 角色2：操作员（Operator）
- **职责**：执行数据爬取任务、数据管理
- **权限**：数据爬取、查询、查看、导出权限
- **主要功能**：
  - 创建爬取任务（手动/定时）
  - 管理爬取任务（暂停、继续、取消）
  - 数据查询与筛选
  - 数据详情查看
  - 数据导出
  - 查看操作日志（仅自己）

#### 角色3：普通用户（User）
- **职责**：查看和分析数据
- **权限**：数据查询、查看、导出权限（只读）
- **主要功能**：
  - 数据查询与筛选
  - 数据详情查看
  - 数据可视化分析
  - 数据导出

### 4.2 用户使用场景

**场景1：操作员创建爬取任务**
- 操作员登录系统
- 进入"爬取管理"页面
- 输入酒店名称，选择平台（可多选）
- 选择爬取模式（立即爬取/定时爬取）
- 提交任务，查看任务状态和进度

**场景2：普通用户查询酒店数据**
- 用户登录系统
- 进入"数据查询"页面
- 设置查询条件（酒店名称、平台、星级、价格范围等）
- 查看查询结果列表
- 点击酒店名称查看详情
- 对比不同平台的数据

**场景3：管理员进行用户管理**
- 管理员登录系统
- 进入"用户管理"页面
- 创建新用户，分配角色和权限
- 查看用户列表，编辑或禁用用户
- 查看用户操作日志

**场景4：数据分析**
- 用户进入"数据分析"页面
- 选择分析维度（价格趋势、点评分析、预订量分析）
- 选择时间范围和平台
- 查看可视化图表
- 导出分析报告

## 五、功能需求详细说明

### 5.1 数据爬取模块

#### 5.1.1 爬取触发功能
- **手动爬取**：
  - 输入酒店名称（支持模糊匹配提示）
  - 选择爬取平台（美团、携程、飞猪、高德，支持多选）
  - 选择爬取类型（仅列表页/列表页+详情页）
  - 点击"开始爬取"按钮
  - 系统验证输入，创建爬取任务

- **定时爬取**：
  - 设置酒店名称（支持多个酒店，每行一个）
  - 选择爬取平台
  - 设置爬取频率（每小时/每天/每周/自定义cron表达式）
  - 设置任务名称和描述
  - 设置任务有效期（开始时间、结束时间）
  - 保存定时任务

#### 5.1.2 爬取过程管理功能
- **任务列表展示**：
  - 显示任务ID、酒店名称、平台、任务类型、创建时间、状态、进度
  - 支持按状态、平台、时间筛选
  - 支持任务搜索

- **任务状态管理**：
  - 等待中：任务已创建，等待执行
  - 爬取中：任务正在执行，显示进度百分比
  - 爬取成功：任务完成，显示爬取数据量
  - 爬取失败：任务失败，显示失败原因
  - 已暂停：任务被暂停
  - 已取消：任务被取消

- **任务操作**：
  - 暂停：暂停正在执行的任务
  - 继续：继续已暂停的任务
  - 取消：取消等待中或已暂停的任务
  - 重试：重新执行失败的任务
  - 查看详情：查看任务执行日志和爬取结果

#### 5.1.3 搜索列表页信息爬取字段
- 酒店名称
- 酒店星级（五星级/四星级/三星级/无星级）
- 点评分数（保留2位小数，记录满分标准）
- 点评条数（支持万、千等单位）
- 酒店起价（包含单位、房型、优惠信息）
- 预订动态（x分钟前有人预订）
- 每天单量（估算或直接获取）
- 平台标识
- 爬取时间

#### 5.1.4 酒店详情页信息爬取字段
- 开业时间（年份+月份，或装修时间）
- 房型列表：
  - 房型名称
  - 房型最低价（包含日期范围）
  - 房型剩余库存（具体数量或范围）
- 平台标识
- 爬取时间

### 5.2 管理平台模块

#### 5.2.1 数据存储与管理功能
- **数据库设计**：
  - 用户表（users）：用户基本信息、角色、权限
  - 爬取任务表（crawl_tasks）：任务信息、状态、进度
  - 酒店列表数据表（hotel_list_data）：搜索列表页数据
  - 酒店详情数据表（hotel_detail_data）：详情页数据
  - 操作日志表（operation_logs）：所有操作记录
  - 系统配置表（system_config）：系统参数配置

- **数据备份**：
  - 自动备份：设置备份周期（每天/每周），自动执行
  - 手动备份：管理员可随时触发备份
  - 备份文件管理：查看备份列表，支持恢复和删除

- **数据清理**：
  - 自动清理：按存储周期自动清理过期数据
  - 手动清理：管理员选择时间段手动清理
  - 清理前确认机制，防止误删

#### 5.2.2 数据查询与展示功能
- **多条件查询**：
  - 酒店名称（支持模糊搜索）
  - 平台名称（多选）
  - 酒店星级（多选）
  - 点评分数范围（最小值-最大值）
  - 价格范围（最小值-最大值）
  - 爬取时间范围（开始时间-结束时间）
  - 预订量范围
  - 高级查询：支持组合条件（AND/OR逻辑）

- **列表展示**：
  - 表格形式展示核心字段
  - 分页功能（10/20/50/100条每页）
  - 排序功能（按价格、评分、预订量、时间等）
  - 批量操作（批量导出、批量删除）
  - 列显示控制（自定义显示列）

- **详情展示**：
  - 酒店基本信息卡片
  - 多平台数据对比（标签页切换）
  - 价格趋势图表
  - 点评趋势图表
  - 房型信息列表
  - 历史数据对比

- **数据可视化**：
  - 价格对比柱状图（多平台）
  - 价格趋势折线图（时间序列）
  - 点评分数分布饼图
  - 预订量热力图
  - 平台数据对比雷达图

#### 5.2.3 数据分析功能
- **价格分析**：
  - 平台价格对比（同一酒店不同平台）
  - 价格趋势分析（按周/月）
  - 价格波动预测
  - 价格优势平台识别

- **点评分析**：
  - 点评分数趋势（时间序列）
  - 关键词提取（词云图）
  - 情感分析（正面/中性/负面）
  - 点评分类统计

- **预订量分析**：
  - 平台预订量对比
  - 时间段预订量分析（高峰时段识别）
  - 预订量趋势预测
  - 热门平台统计

#### 5.2.4 数据导出功能
- **导出范围选择**：
  - 当前查询结果
  - 指定时间段数据
  - 指定平台数据
  - 指定酒店数据
  - 自定义筛选条件

- **导出格式**：
  - Excel（.xlsx）：支持多工作表，格式美化
  - CSV（.csv）：纯文本格式，兼容性好
  - JSON（.json）：结构化数据，便于程序处理

- **导出功能**：
  - 显示导出进度
  - 导出完成后自动下载
  - 导出历史记录
  - 支持重新下载已导出文件
  - 导出文件有效期管理

#### 5.2.5 用户管理功能（仅管理员）
- **用户创建**：
  - 用户名（唯一性校验）
  - 密码（复杂度校验）
  - 姓名
  - 联系方式
  - 邮箱（格式校验）
  - 角色选择（普通用户/操作员/管理员）
  - 状态（启用/禁用）

- **权限分配**：
  - 角色权限矩阵配置
  - 自定义权限组
  - 用户级权限覆盖
  - 权限继承机制

- **用户信息管理**：
  - 用户列表展示（分页、搜索、筛选）
  - 用户详情查看
  - 用户信息编辑
  - 密码重置
  - 账号启用/禁用
  - 用户操作日志查看

#### 5.2.6 系统设置功能（仅管理员）
- **爬取设置**：
  - 默认爬取超时时间（秒）
  - 请求间隔时间（秒）
  - 重试次数
  - 并发爬取数量
  - User-Agent池配置
  - 代理IP配置（可选）

- **存储设置**：
  - 数据默认存储周期（月）
  - 备份文件存储路径
  - 备份文件保留时间（天）
  - 数据归档策略

- **日志管理**：
  - 日志级别设置（DEBUG/INFO/WARN/ERROR）
  - 日志存储周期（天）
  - 日志查询（时间、用户、操作类型、关键词）
  - 日志导出
  - 日志清理策略

## 六、非功能需求

### 6.1 性能需求
- **爬取性能**：
  - 单条酒店单平台爬取时间 ≤ 30秒
  - 10条酒店（每平台10条）总时间 ≤ 5分钟
  - 支持并发爬取（最多10个并发任务）

- **查询性能**：
  - 简单查询响应时间 ≤ 2秒
  - 复杂查询响应时间 ≤ 5秒
  - 支持数据库索引优化

- **并发性能**：
  - 支持至少50个用户同时在线
  - 系统响应时间 ≤ 3秒
  - 无明显卡顿或延迟

### 6.2 可靠性需求
- **数据可靠性**：
  - 爬取数据准确率 ≥ 95%
  - 数据修正功能
  - 数据校验机制

- **系统可靠性**：
  - MTBF ≥ 1000小时
  - MTTR ≤ 30分钟
  - 数据不丢失保证
  - 异常恢复机制

### 6.3 安全性需求
- **用户认证**：
  - 用户名密码认证
  - 密码加密存储（SHA-256）
  - 密码复杂度校验（8位以上，包含大小写字母、数字、特殊字符）
  - 登录失败锁定机制（5次失败锁定30分钟）
  - 会话超时机制（30分钟无操作自动登出）

- **权限控制**：
  - 基于角色的访问控制（RBAC）
  - 接口级权限验证
  - 前端路由权限控制
  - 防止越权操作

- **数据安全**：
  - HTTPS传输加密
  - 数据库定期备份
  - 敏感数据脱敏显示
  - SQL注入防护
  - XSS攻击防护

- **反爬应对**：
  - 模拟浏览器请求头
  - 动态请求间隔
  - 代理IP池（可选）
  - 验证码识别（OCR/人工）

### 6.4 易用性需求
- **界面设计**：
  - 简洁直观的界面风格
  - 符合用户习惯的操作流程
  - 响应式设计（支持PC、平板）
  - 无障碍访问支持

- **操作指引**：
  - 关键操作帮助提示
  - 操作指引文档
  - 新手引导流程
  - 错误提示清晰明确

### 6.5 可扩展性需求
- **平台扩展**：
  - 插件化爬取模块
  - 配置化平台参数
  - 易于新增平台

- **功能扩展**：
  - 模块化架构设计
  - API接口标准化
  - 第三方系统对接接口

- **性能扩展**：
  - 支持水平扩展
  - 缓存机制（Redis）
  - 数据库读写分离
  - 负载均衡支持
