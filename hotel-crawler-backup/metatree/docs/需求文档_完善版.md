# 酒店信息爬取及管理平台 - 完善需求文档

## 一、项目背景
随着旅游行业的蓬勃发展，用户对于酒店信息的需求日益多样化，需要便捷地获取不同平台（美团、携程、飞猪、高德）上酒店的全面信息，以便进行对比和选择。同时，企业也需要一个高效的管理平台对爬取到的酒店信息进行统一管理、分析和应用。为满足这一需求，特规划开发本酒店信息爬取及管理平台，实现对主流平台酒店信息的精准爬取与高效管理。

## 二、产品目标
1. 实现对美团、携程、飞猪、高德四大平台酒店信息的自动化爬取，涵盖搜索列表页和酒店详情页指定字段信息。
2. 搭建功能完善的管理平台，支持对爬取数据的存储、查询、展示、分析、导出等操作，为用户提供全面的酒店信息管理服务。
3. 保证爬取数据的准确性、及时性和完整性，确保管理平台运行稳定、操作便捷，满足用户对酒店信息管理的多样化需求。

## 三、产品范围
1. **信息爬取范围**：覆盖美团、携程、飞猪、高德四大主流酒店预订平台，针对用户输入的酒店名称，爬取该酒店在各平台的搜索列表页和详情页指定字段信息。
2. **管理平台功能范围**：包括数据爬取管理、数据存储与管理、数据查询与展示、数据分析、数据导出、用户管理、系统设置等模块。

## 五、功能需求详细设计

### 5.1 数据爬取模块

#### 5.1.1 爬取架构设计

**架构说明**：
系统采用 **LangGraph** 框架构建爬取流程，将爬取任务分解为多个角色（Agent）协作完成。每个平台拥有独立的流程图，通过状态机管理爬取流程。

**角色定义**：
1. **网址获取角色 (URLFetcherAgent)**: 获取平台URL配置
2. **登入角色 (LoginAgent)**: 处理登录流程
3. **定位角色 (LocatorAgent)**: 定位页面元素
4. **酒店查找角色 (SearchAgent)**: 执行搜索操作
5. **数据提取角色 (ExtractorAgent)**: 提取结构化数据
6. **内容校验角色 (ValidatorAgent)**: 校验数据质量
7. **错误处理角色 (ErrorHandlerAgent)**: 处理异常情况

**流程特点**：
- 每个平台独立流程，互不干扰
- 支持流程可视化监控
- 支持暂停、恢复、重试
- 支持错误自动处理和回退

#### 5.1.2 爬取任务创建
**功能描述**：
- 支持手动创建爬取任务
- 支持定时爬取任务设置
- 支持批量酒店名称导入
- 支持多平台并行爬取

**输入字段**：
- 酒店名称（必填，支持模糊匹配）
- 目标平台（必填，多选：美团、携程、飞猪、高德）
- 爬取类型（单选：仅列表页、仅详情页、全部）
- 任务类型（单选：立即执行、定时执行）
- 定时设置（如选择定时执行）：执行频率（每小时/每天/每周）、执行时间、结束时间
- 登录凭证（可选，如需要登录的平台）

**交互流程**：
1. 用户进入"爬取任务"页面
2. 点击"新建任务"按钮
3. 填写任务信息表单
4. 选择目标平台（系统自动加载对应平台的流程图）
5. 点击"提交"创建任务
6. 系统返回任务ID和预计执行时间
7. 系统启动LangGraph流程执行爬取

#### 5.1.2 爬取任务管理
**功能描述**：
- 任务列表展示（支持筛选和搜索）
- 任务状态实时更新
- 任务操作（暂停、继续、取消、重试）
- 任务详情查看
- 流程可视化监控（显示LangGraph流程图和当前节点）

**任务状态**：
- 等待中（Pending）
- 爬取中（Running）- 显示当前执行的节点
- 已完成（Completed）
- 已失败（Failed）
- 已暂停（Paused）
- 已取消（Cancelled）

**任务列表字段**：
- 任务ID
- 酒店名称
- 目标平台
- 创建时间
- 执行时间
- 状态（包含当前节点信息）
- 进度（百分比，基于已完成的节点数）
- 操作按钮

**流程监控**：
- 显示当前执行的节点
- 显示已完成的节点列表
- 显示节点执行时间
- 显示错误节点和错误信息

#### 5.1.3 爬取数据字段定义

**搜索列表页字段**：
1. 酒店名称（hotel_name）
2. 酒店星级（star_level）
3. 点评分数（rating_score）
4. 点评条数（review_count）
5. 酒店起价（min_price）
6. 预订动态（booking_dynamic）
7. 平台名称（platform）
8. 爬取时间（crawl_time）

**详情页字段**：
1. 开业时间（opening_date）
2. 房型列表（room_types）：房型名称、最低价、剩余库存
3. 酒店地址（address）
4. 酒店图片（images，可选）
5. 酒店设施（facilities，可选）

### 5.2 数据管理模块

#### 5.2.1 数据查询
**查询条件**：
- 酒店名称（支持模糊搜索）
- 平台（多选）
- 星级（多选）
- 点评分数范围（最小值-最大值）
- 价格范围（最小值-最大值）
- 爬取时间范围（开始时间-结束时间）
- 城市（可选）

**查询结果展示**：
- 列表模式：表格展示核心字段
- 卡片模式：卡片式展示，更直观
- 详情模式：点击查看完整信息

**排序功能**：
- 按价格升序/降序
- 按点评分数升序/降序
- 按爬取时间升序/降序
- 按预订量升序/降序

**详情页内容**：
- 酒店基本信息（名称、星级、地址）
- 各平台数据对比（标签页切换）
- 价格趋势图表
- 点评趋势图表
- 预订量统计
- 房型详情列表

#### 5.2.3 数据分析
**分析维度**：
1. **价格分析**
   - 平台价格对比（柱状图）
   - 价格趋势（折线图）
   - 价格分布（箱线图）

2. **点评分析**
   - 点评分数趋势（折线图）
   - 点评关键词云（词云图）
   - 情感分析（饼图：正面/中性/负面）

3. **预订量分析**
   - 平台预订量对比（柱状图）
   - 预订量趋势（折线图）
   - 预订高峰时段（热力图）

## 七、UI/UX设计规范

### 7.1 设计风格
- **风格定位**：现代简约、专业高效
- **色彩主调**：蓝色系（专业、可信赖）
- **辅助色彩**：绿色（成功）、橙色（警告）、红色（错误）、灰色（中性）

### 7.2 色彩系统
- **主色**：#1890FF（蓝色）
- **成功色**：#52C41A（绿色）
- **警告色**：#FAAD14（橙色）
- **错误色**：#F5222D（红色）
- **文字色**：
  - 主文字：#262626
  - 次文字：#595959
  - 辅助文字：#8C8C8C
- **背景色**：
  - 主背景：#F0F2F5
  - 卡片背景：#FFFFFF
  - 边框：#D9D9D9

### 7.3 字体规范
- **字体族**：-apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', sans-serif
- **字号**：
  - 大标题：24px
  - 标题：20px
  - 副标题：16px
  - 正文：14px
  - 辅助文字：12px

### 7.4 间距规范
- **基础间距单位**：8px
- **常用间距**：8px, 16px, 24px, 32px, 48px

### 7.5 组件规范
- **按钮**：
  - 主要按钮：主色背景，白色文字
  - 次要按钮：白色背景，主色边框和文字
  - 文字按钮：无背景，主色文字
- **输入框**：圆角4px，边框1px
- **卡片**：圆角8px，阴影：0 2px 8px rgba(0,0,0,0.1)
- **表格**：斑马纹，悬停高亮

### 7.6 交互规范
- **加载状态**：显示加载动画或骨架屏
- **空状态**：显示友好的空状态插图和提示文字
- **错误提示**：顶部通知栏或弹窗提示
- **成功提示**：绿色通知栏提示
- **确认操作**：危险操作需二次确认（弹窗）

## 八、非功能需求补充

### 8.1 性能需求
- 页面首次加载时间 < 3秒
- 接口响应时间 < 2秒（简单查询）
- 支持至少50并发用户
- 数据导出（1000条） < 10秒

### 8.2 兼容性需求
- 浏览器支持：Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- 响应式设计：支持1920x1080及以上分辨率，最小支持1366x768

### 8.3 可访问性需求
- 支持键盘导航
- 重要操作提供快捷键
- 符合WCAG 2.1 AA级标准

- POST /api/auth/login
- POST /api/auth/logout
- GET /api/auth/me

- GET /api/tasks（任务列表）

- GET /api/settings
